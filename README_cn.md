# 芯控家居公开框架

[English README](README.md)

`Xinkong Home` 是一套围绕 RDK X5 构建的边缘 AIoT 家庭控制框架。这个公开仓库是面向比赛评审和技术展示整理出的框架快照：它展示系统的主要架构、模块边界和扩展点，但不会公开私有部署细节，也不会公开完整的比赛实现。

中文名称：`芯控家居`

## 这个仓库展示了什么

- 以语音为入口的交互流程，并带有唤醒词门控。
- 将自然语言指令路由为结构化任务的意图识别流程。
- 一个中央编排器，用于协调智能家居控制、本地边缘动作、摄像头/BPU 视觉、健康传感器、天气查询和语音反馈。
- Home Assistant、OpenClaw、ESP32/MQTT 设备、RDK X5 BPU 推理和本地 TTS 的适配器边界。
- 面向边缘部署的运行状态检查和容错回退概念。
- 一个可运行的离线模拟示例，用于展示控制流程，不依赖真实密钥、模型或设备。

## 比赛展示亮点

- **RDK X5 边缘中枢：** 系统围绕 RDK X5 边缘节点组织，将摄像头/BPU 感知、语音交互、本地服务和智能家居控制放在同一个运行框架中。
- **多模态闭环：** 语音指令可以通过统一编排层触发设备控制、本地视觉检查、健康检测、天气查询和系统状态检查。
- **BPU-aware 设计：** 公开框架展示 BPU 视觉模块边界，但不公开模型文件、预处理细节和私有推理调优。
- **安全优先的编排：** 高风险动作在适配器执行前会经过风险策略边界；私有的场景融合规则不会在本仓库公开。
- **混合可靠性：** 完整项目结合在线 AI 服务、本地兜底、启动预热、看门狗检查和健康快照，以适应比赛现场环境。
- **清晰的扩展点：** 通过适配器隔离 Home Assistant、OpenClaw、ESP32/MQTT、健康传感器、天气、TTS 和视觉模块，便于替换与扩展。

## 哪些内容没有公开

完整比赛项目包含硬件参数、生产配置、模型文件、私有恢复逻辑、现场调优的安全策略以及完整编排代码。这些内容不包含在当前公开仓库中。

公开版会用接口、简化示例逻辑和注释替代敏感或高价值实现。具体来说，本仓库不包含：

- API key、token、热点密码、真实网络地址或 `.env` 文件。
- BPU `.bin` 模型、ONNX 模型、Vosk 模型、Piper 声音模型或模型转换脚本。
- 完整生产版 `main.py` 逻辑。
- 完整 Home Assistant/OpenClaw 白名单配置。
- 比赛现场使用的完整 ESP32 固件。
- 比赛报告、演示脚本、私有日志、人脸库或运行数据。

## 高层流程

```text
语音/文本输入
  -> 唤醒词门控和会话保护
  -> 意图路由
  -> 任务编排器
  -> 安全策略
  -> 适配器：Home Assistant / OpenClaw / 摄像头+BPU / 健康 / 天气
  -> 回复组装
  -> 本地或云端 TTS
```

## 快速开始

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install -e ".[dev]"
python main.py --demo
PYTHONPATH=src python -m unittest discover -s tests
```

这个 demo 只使用模拟适配器，不会连接 Home Assistant、MQTT、OpenClaw、云端语音服务或 RDK 硬件。

## 公开版 `main.py` 的边界

公开版 `main.py` 保留了启动流程的形状：配置加载、模块构建、事件处理、任务规划、安全检查、适配器分发和回复生成。

生产细节会以注释形式保留位置，但不会公开完整实现，包括：

- 实时云端语音会话监管；
- 嘈杂环境下的唤醒词调优；
- RDK X5 开机优化和模块预热调度；
- 硬件相关的 BPU 推理调用；
- 安全场景融合逻辑；
- 比赛演示恢复路径。
